### P0 — Critical (security)

#### [P0-1] Self-signed cert installed into Trusted Root + hardcoded PFX password — **S–M**
**Evidence (verified):**
- `sign.ps1:42` → `$PFX_PASSWORD = "sidearm-dev"` (plaintext, in repo).
- `sign.ps1:201-208` → `New-SelfSignedCertificate -Type CodeSigningCert`, 10-year validity (`$CERT_YEARS = 10`, `sign.ps1:41`).
- `sign.ps1:228` → `certutil -addstore -f "Root" $CER_PATH` — installs into **Trusted Root** store.
- `src-tauri/tauri.conf.json:41` bundles `../resources/sidearm-ca.cer`; `tauri.conf.json:47` runs NSIS `installerHooks` (`./nsis-hooks.nsh`) with `installMode: perMachine` → the cert is installed on **every end-user machine**.
- Cross-ref: `CODEX_REVIEW_2026-04-26.md` §High.

**Why it matters:** to let the app inject input into elevated windows (`uiAccess`, enabled at build via `SIDEARM_UIACCESS`), Windows requires a signature chaining to a trusted root. Sidearm satisfies this by making *its own* cert a trusted root on users' machines. The private key (PFX) is protected only by the password `"sidearm-dev"`, which is public (it's in the script). If the PFX ever leaks (it is exported to `certs/` "for CI/archive"), an attacker can sign **arbitrary code that every Sidearm user's Windows will trust for 10 years**, bypassing SmartScreen. For an input-synthesis app this is a high-value target.

**Fix (choose one):**
1. **Proper:** buy an EV/OV code-signing cert (DigiCert/Sectigo). Only legitimate way to get `uiAccess` without installing a custom root. ~$200–500/yr.
2. **If `uiAccess` isn't essential for most users:** remove `uiAccess` from the manifest, delete the CA install from the NSIS hook and the bundled `sidearm-ca.cer` resource, document the UIPI limitation. Zero cost, zero risk.
3. **If keeping dev signing temporarily:** (a) remove the `certutil -addstore "Root"` step from the NSIS hook + drop the bundled `.cer` resource — never touch end-user trust stores; (b) move the PFX password to an env var / secret store; (c) shorten validity to ~1–2 years; (d) ensure `certs/*.pfx` is git-ignored and purge it from history if ever committed.

**Verify:** a fresh install adds nothing to `LocalMachine\Root`; no plaintext signing password remains in the repo; `git log -- certs/` shows no PFX.

### P1 — High

#### [P1-1] Action conditions are saved but never enforced — **S**
**Evidence (verified):**
- `src-tauri/src/executor.rs:991-992` → `pub fn evaluate_conditions(...)` is correct **but marked `#[allow(dead_code)]`** — i.e. never called anywhere.
- `src-tauri/src/resolver.rs:197-202` → resolves the action from `binding.action_ref` and returns `Resolved` **without checking `action.conditions`**.
- Cross-ref: `CODEX_REVIEW_2026-04-26.md` §Medium 1.

**Why it matters:** the UI lets users attach conditions ("run only when `excel.exe` is active", "only if the window title contains …"). They are persisted to config but the engine ignores them and **always executes**. A destructive macro guarded by a condition fires everywhere — a correctness *and* safety bug.

**Fix:**
1. Call `evaluate_conditions(&action.conditions, exe, title)` in the **execution** path, and in `resolve_input_preview` so diagnostics show why an action didn't run.
2. Thread the active-window context (`exe`, `title`) to the check — it is already obtained for profile resolution (`profile_resolution`); extend it to the action step.
3. Remove `#[allow(dead_code)]` from `evaluate_conditions`.
4. Add unit tests: `ExeEquals` match/mismatch, `ExeNotEquals`, `WindowTitleContains`/`NotContains`, and empty-conditions ⇒ pass (already implemented).
5. Add a `ConditionUnmet` preview status so the diagnostics panel explains skips.

**Verify:** a binding with `ExeEquals = "excel.exe"` does not fire in other apps; tests cover all `ActionCondition` variants.

#### [P1-2] No CI gate on pull requests — **M**
**Evidence (verified):** `.github/workflows/audit.yml` is the only workflow — weekly `cargo audit` + `npm audit` (cron `0 9 * * 1`). No `cargo check/clippy/test`, `tsc --noEmit`, or `vitest` runs on PRs. (CODEX review notes tests were already failing on the audit machine — a gate would have caught it.)

- Fold `cargo audit` / `npm audit` into this PR gate.
- Add **gitleaks** (secret scan) and **cargo-deny** (advisories/licenses/bans) jobs.

See §2 item B18 for the full donor CI config to adapt.

**Verify:** a PR with a clippy warning or failing test is blocked; gitleaks flags the `sidearm-dev` password if P0-1 isn't done yet.

### P2 — Medium

#### [P2-1] Synapse import has no size/decompression caps (zip-bomb) — **S**
**Evidence (verified):** `src-tauri/src/synapse_import/format_v3.rs:56` `zip::ZipArchive::new(reader)` then per-entry `read_file(...)` decompresses with no byte cap (`zip = "0.6"`, `Cargo.toml:41`). CODEX §Medium 2.
**Fix:** cap entry count and total/per-entry decompressed bytes; reject archives that exceed limits before reading.

#### [P2-2] Generic file IPC broader than needed — **S**
**Evidence (verified):** `src-tauri/src/lib.rs:329-358` — `write_text_file` / `read_text_file` accept any absolute path under `$USERPROFILE` (`validate_user_file_path`, `lib.rs:239-315`). CODEX §Medium 5.
**Why it matters:** a compromised renderer could read/write shell profiles or startup files anywhere in the home dir.
**Fix:** scope these commands to specific export/import directories (e.g. an app-owned subfolder), or pass an operation kind that restricts the allowed base directory.

#### [P2-3] CSP allows `'unsafe-inline'` styles — **S**
**Evidence (verified):** `src-tauri/tauri.conf.json:27` → `style-src 'self' 'unsafe-inline'`.
**Fix:** drop `'unsafe-inline'` (move inline styles to CSS files, or use nonces/hashes) to harden the renderer.

#### [P2-4] No release-profile tuning — **S** (pattern from sibling *Sweet Whisper*)
**Evidence (verified):** `Cargo.toml` defines only `[profile.dev]` (lines 87-91); the default release profile is used (`lto=false`, `codegen-units=16`, `panic=unwind`).
**Fix:** add a `[profile.release]` block (see §2 item C8 for the exact donor TOML):
```toml
[profile.release]
lto = true
codegen-units = 1
strip = "debuginfo"
panic = "abort"      # caveat: confirm no std::panic::catch_unwind reliance first
opt-level = "s"      # size; use 3 if hook-callback latency profiling demands speed
```
Smaller binary, less unwinding machinery (reduced attack surface for an input-synthesis app).

#### [P2-5] No cross-platform build entry / CI-runnable build — **M** (pattern from *Sweet Whisper*)
**Evidence (verified):** packaging is `build_portable.ps1` (PowerShell only); no `xtask`. There is a `platform/linux` stub but no build path for it.
**Fix:** add an `xtask/` workspace member exposing `cargo xtask dev|check|test|dist` (port the per-OS `CARGO_TARGET_DIR`, target-dir override, and build steps). Gives a single OS-agnostic entry point usable from CI (P1-2). See §2 item C8 for the full donor implementation.

#### [P2-6] Settings schema not a single source of truth — **M** (pattern from *Sweet Whisper*)
**Evidence:** Rust `serde` config models and the TS `backend.ts` types are maintained by hand; risk of snake_case/camelCase drift.
**Fix:** make `schemas/config.v2.schema.json` (or the Rust models) the source, generate TS types from it, and add a CI drift check.

### [C8] cargo-xtask — кросс-платформенный оркестратор сборки

**Что/зачем.** Sidearm собирается только через `build_portable.ps1` (PowerShell), что делает CI неудобным и ломает на Linux. Sweet Whisper решает это через `cargo xtask` — zero-dep Rust бинарник, единая точка входа для `dev`, `check`, `test`, `dist`. Ключевое: он устанавливает `CARGO_TARGET_DIR` через env var (не `.cargo/config.toml`, чтобы избежать tauri#4632), что на Windows убирает проблему MAX_PATH-260 при глубоких зависимостях.

**Куда класть:**
```
xtask/
  src/
    main.rs        ← файл ниже
  Cargo.toml       ← [package] name="xtask", edition="2021"
```
В корневом `Cargo.toml` добавить `"xtask"` в `[workspace] members`.

**Донорский код (Sweet Whisper `xtask/src/main.rs`, полностью):**

```rust
//! Sidearm build task runner (cargo-xtask pattern).
//!
//! Usage: `cargo xtask <command>`
//!
//! Commands:
//!   dev     — Start Tauri dev server
//!   check   — Run all quality checks: fmt, clippy, tests, frontend
//!   test    — Run Rust + frontend tests
//!   dist    — Delegate to build_portable.ps1 (Windows only)

use std::env;
use std::path::{Path, PathBuf};
use std::process::{Command, ExitCode};

fn main() -> ExitCode {
    let args: Vec<String> = env::args().collect();
    if args.len() < 2 {
        print_usage();
        return ExitCode::FAILURE;
    }
    let project_root = project_root();
    match args[1].as_str() {
        "dev"  => cmd_dev(&project_root),
        "check"=> cmd_check(&project_root),
        "test" => cmd_test(&project_root),
        "dist" => cmd_dist(&project_root),
        "help" | "--help" | "-h" => { print_usage(); ExitCode::SUCCESS }
        other => {
            eprintln!("error: unknown command '{other}'");
            eprintln!();
            print_usage();
            ExitCode::FAILURE
        }
    }
}

fn print_usage() {
    eprintln!("\
Sidearm xtask — cross-platform build orchestration

Usage: cargo xtask <command>

Commands:
  dev     Start Tauri dev server with hot reload
  check   Run all quality checks (fmt, clippy, tests, frontend)
  test    Run Rust + frontend tests only
  dist    Build production portable release (Windows, delegates to ps1)");
}

// ---------------------------------------------------------------------------
// Project root detection
// ---------------------------------------------------------------------------
fn project_root() -> PathBuf {
    let xtask_dir =
        env::var("CARGO_MANIFEST_DIR").unwrap_or_else(|_| env!("CARGO_MANIFEST_DIR").to_string());
    Path::new(&xtask_dir)
        .parent()
        .expect("xtask must be inside the workspace")
        .to_path_buf()
}

// ---------------------------------------------------------------------------
// Platform-specific CARGO_TARGET_DIR
// ---------------------------------------------------------------------------
/// Apply CARGO_TARGET_DIR to avoid MAX_PATH-260 on Windows (Cargo issue #8801).
/// Keep root ≤15 chars including drive letter: deep build artifacts can reach
/// ~240 chars under the target dir and overrun MAX_PATH on MSBuild/FileTracker.
/// Override with SIDEARM_TARGET_DIR env var.
fn apply_target_dir(cmd: &mut Command) {
    if let Ok(custom) = env::var("SIDEARM_TARGET_DIR") {
        cmd.env("CARGO_TARGET_DIR", custom);
    } else if cfg!(windows) {
        // Short off-system path — adjust if D:\ is preferred
        cmd.env("CARGO_TARGET_DIR", r"C:\sd-build");
    } else {
        cmd.env("CARGO_TARGET_DIR", "/tmp/sd-target");
    }
}

// ---------------------------------------------------------------------------
// Optional accelerators: sccache and cargo-nextest
// ---------------------------------------------------------------------------
fn apply_compile_cache(cmd: &mut Command) {
    if has_executable("sccache") {
        cmd.env("RUSTC_WRAPPER", "sccache");
    }
}

fn test_subcommand() -> &'static [&'static str] {
    if has_executable("cargo-nextest") {
        &["nextest", "run"]
    } else {
        &["test"]
    }
}

fn has_executable(name: &str) -> bool {
    let exe = if cfg!(windows) { format!("{name}.exe") } else { name.to_string() };
    env::var_os("PATH")
        .map(|paths| env::split_paths(&paths).any(|p| p.join(&exe).is_file()))
        .unwrap_or(false)
}

// ---------------------------------------------------------------------------
// Helper: run a command
// ---------------------------------------------------------------------------
fn run(cmd: &mut Command) -> ExitCode {
    match cmd.status() {
        Ok(status) if status.success() => ExitCode::SUCCESS,
        Ok(status) => ExitCode::from(status.code().unwrap_or(1) as u8),
        Err(e) => {
            eprintln!("error: failed to run {:?}: {e}", cmd.get_program());
            ExitCode::FAILURE
        }
    }
}

fn run_ok(cmd: &mut Command) -> bool {
    matches!(cmd.status(), Ok(s) if s.success())
}

// ---------------------------------------------------------------------------
// Subcommands
// ---------------------------------------------------------------------------
fn cmd_dev(root: &Path) -> ExitCode {
    eprintln!("=== cargo xtask dev ===");
    let mut cmd = Command::new("cargo");
    cmd.current_dir(root).args(["tauri", "dev"]);
    apply_target_dir(&mut cmd);
    apply_compile_cache(&mut cmd);
    run(&mut cmd)
}

fn cmd_check(root: &Path) -> ExitCode {
    eprintln!("=== cargo xtask check ===");
    let tauri_dir = root.join("src-tauri");
    let ui_dir = root.join("src");           // Sidearm frontend dir
    let mut failed: Vec<&str> = Vec::new();

    eprintln!("\n--- [1/5] Rust fmt ---");
    {
        let mut cmd = Command::new("cargo");
        cmd.current_dir(root)
            .args(["fmt", "--all", "--manifest-path"])
            .arg(tauri_dir.join("Cargo.toml"))
            .args(["--", "--check"]);
        apply_target_dir(&mut cmd);
        if !run_ok(&mut cmd) { failed.push("rust-fmt"); }
    }

    eprintln!("\n--- [2/5] Rust clippy ---");
    {
        let mut cmd = Command::new("cargo");
        cmd.current_dir(root)
            .args(["clippy", "--lib", "--manifest-path"])
            .arg(tauri_dir.join("Cargo.toml"))
            .args(["--", "-D", "warnings"]);
        apply_target_dir(&mut cmd);
        apply_compile_cache(&mut cmd);
        if !run_ok(&mut cmd) { failed.push("rust-clippy"); }
    }

    eprintln!("\n--- [3/5] Rust tests ---");
    {
        let mut cmd = Command::new("cargo");
        cmd.current_dir(root)
            .args(test_subcommand())
            .args(["--lib", "--manifest-path"])
            .arg(tauri_dir.join("Cargo.toml"));
        apply_target_dir(&mut cmd);
        apply_compile_cache(&mut cmd);
        if !run_ok(&mut cmd) { failed.push("rust-tests"); }
    }

    eprintln!("\n--- [4/5] Frontend type-check ---");
    if ui_dir.join("node_modules").is_dir() {
        let mut cmd = npm_cmd();
        cmd.current_dir(root).args(["run", "typecheck"]);
        if !run_ok(&mut cmd) { failed.push("frontend-typecheck"); }
    } else {
        eprintln!("  SKIP: node_modules not found (run npm install)");
    }

    eprintln!("\n--- [5/5] Frontend tests ---");
    if ui_dir.join("node_modules").is_dir() {
        let mut cmd = npm_cmd();
        cmd.current_dir(root).args(["run", "test"]);
        if !run_ok(&mut cmd) { failed.push("frontend-tests"); }
    } else {
        eprintln!("  SKIP: node_modules not found");
    }

    eprintln!();
    if failed.is_empty() {
        eprintln!("All checks passed.");
        ExitCode::SUCCESS
    } else {
        eprintln!("FAILED: {}", failed.join(", "));
        ExitCode::FAILURE
    }
}

fn cmd_test(root: &Path) -> ExitCode {
    eprintln!("=== cargo xtask test ===");
    let tauri_dir = root.join("src-tauri");
    let mut failed: Vec<&str> = Vec::new();

    eprintln!("\n--- Rust tests ---");
    {
        let mut cmd = Command::new("cargo");
        cmd.current_dir(root)
            .args(test_subcommand())
            .args(["--lib", "--manifest-path"])
            .arg(tauri_dir.join("Cargo.toml"));
        apply_target_dir(&mut cmd);
        apply_compile_cache(&mut cmd);
        if !run_ok(&mut cmd) { failed.push("rust-tests"); }
    }

    eprintln!("\n--- Frontend tests ---");
    {
        let mut cmd = npm_cmd();
        cmd.current_dir(root).args(["run", "test"]);
        if !run_ok(&mut cmd) { failed.push("frontend-tests"); }
    }

    if failed.is_empty() { eprintln!("\nAll tests passed."); ExitCode::SUCCESS }
    else { eprintln!("\nFAILED: {}", failed.join(", ")); ExitCode::FAILURE }
}

fn cmd_dist(root: &Path) -> ExitCode {
    eprintln!("=== cargo xtask dist ===");
    if !cfg!(windows) {
        eprintln!("dist is currently Windows-only (delegates to build_portable.ps1)");
        return ExitCode::FAILURE;
    }
    let script = root.join("build_portable.ps1");
    if !script.exists() {
        eprintln!("error: build_portable.ps1 not found at {}", script.display());
        return ExitCode::FAILURE;
    }
    let mut cmd = Command::new("powershell.exe");
    cmd.current_dir(root)
        .args(["-ExecutionPolicy", "Bypass", "-File"])
        .arg(&script);
    let args: Vec<String> = env::args().skip(2).collect();
    for arg in &args { cmd.arg(arg); }
    run(&mut cmd)
}

fn npm_cmd() -> Command {
    if cfg!(windows) { Command::new("npm.cmd") } else { Command::new("npm") }
}
```

### [B18] 4-джоб CI + cargo-deny + Dependabot

**Что/зачем.** Единственный workflow Sidearm — `audit.yml` (еженедельно, без PR-гейта). Тесты падали на машине CI-аудита, и никто не знал. Sweet Whisper даёт эталонный 4-джоб CI: `repo-hygiene` → `quality` (Windows, xtask check + cargo-deny + cargo-audit + drift-gate) → `quality-linux` (Ubuntu, clippy/test) → `e2e-smoke` (Playwright). Плюс Dependabot на 4 экосистемы.

**Куда класть:**
```
.github/
  workflows/
    ci.yml           ← заменить/дополнить audit.yml
  dependabot.yml     ← новый файл
src-tauri/
  deny.toml          ← новый файл
```

**`.github/workflows/ci.yml` (адаптировано из Sweet Whisper):**
```yaml
name: CI

on:
  push:
    branches: [main, master]
  pull_request:
  workflow_dispatch:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

env:
  # Обновлять здесь при смене toolchain (и в rust-toolchain.toml если он есть)
  RUST_TOOLCHAIN: "1.86.0"   # подставить актуальную версию Sidearm

jobs:
  repo-hygiene:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Ensure build artifacts are not tracked
        shell: bash
        run: |
```

```rust
use serde_json::Value;
use std::path::PathBuf;

/// Macro-generated SidearmCommand enum.
///
/// Добавление новой команды ТРЕБУЕТ добавления варианта здесь.
/// Компилятор затем форсирует exhaustive match в validate_params() и router —
/// ни одна новая команда не проскочит без явной валидации.
macro_rules! sidearm_commands {
    ($( $variant:ident => $str:literal ),* $(,)?) => {
        #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
        pub(crate) enum SidearmCommand {
            $( $variant, )*
        }

        impl SidearmCommand {
            /// Parse command string → enum (acts as whitelist).
            /// Returns None for unrecognized commands.
            pub(crate) fn from_str(s: &str) -> Option<Self> {
                match s {
                    $( $str => Some(Self::$variant), )*
                    _ => None,
                }
            }

            #[allow(dead_code)]
            pub(crate) fn as_str(self) -> &'static str {
                match self {
                    $( Self::$variant => $str, )*
                }
            }
        }
    };

static ALLOWED_PATH_ROOTS: LazyLock<Vec<PathBuf>> = LazyLock::new(|| {
    let mut roots = Vec::new();

    // User home directory
    if let Some(home) = std::env::var_os("USERPROFILE").or_else(|| std::env::var_os("HOME")) {
        if let Ok(canonical) = std::fs::canonicalize(&home) {
            roots.push(canonical);
        } else {
            roots.push(PathBuf::from(home));
        }
    }

    // System temp directory
    let temp = std::env::temp_dir();
    if let Ok(canonical) = std::fs::canonicalize(&temp) {
        roots.push(canonical);
    } else {
        roots.push(temp);
    }

    // Executable directory (portable mode)
    if let Ok(exe_path) = std::env::current_exe() {
        if let Some(exe_dir) = exe_path.parent() {
            if let Ok(canonical) = std::fs::canonicalize(exe_dir) {
                roots.push(canonical);
            } else {
                roots.push(exe_dir.to_path_buf());
            }
        }
    }

    // Current working directory (development)
    if let Ok(cwd) = std::env::current_dir() {
        if let Ok(canonical) = std::fs::canonicalize(&cwd) {
            roots.push(canonical);
        } else {
            roots.push(cwd);
        }
    }

    log::debug!("[SECURITY] Allowed path roots: {:?}",
        roots.iter().map(|r| r.display().to_string()).collect::<Vec<_>>());
    roots
});

/// Validate path using canonicalize + allowed-roots prefix check.
/// Prevents path traversal: `..`, symlinks, absolute escapes.
/// For files that don't exist yet — canonicalizes the parent.
fn validate_path_security(path: &str) -> Result<(), String> {
    if path.is_empty() {
        return Err("Invalid path: path cannot be empty".into());
    }

    let input_path = std::path::Path::new(path);

    let canonical = if let Ok(c) = std::fs::canonicalize(input_path) {
        c
    } else {
        let parent = input_path.parent()
            .ok_or_else(|| format!("Invalid path: cannot determine parent for '{path}'"))?;
        let file_name = input_path.file_name()
            .ok_or_else(|| format!("Invalid path: cannot determine file name for '{path}'"))?;
        let canonical_parent = std::fs::canonicalize(parent)
            .map_err(|_| format!("Invalid path: parent dir does not exist for '{path}'"))?;
        canonical_parent.join(file_name)
    };

    let allowed = ALLOWED_PATH_ROOTS.iter().any(|root| canonical.starts_with(root));

    if !allowed {
        log::warn!("[SECURITY] Path outside allowed roots blocked: '{}' (resolved: '{}')",
            path, canonical.display());
        return Err(format!("Access denied: path '{path}' is outside allowed directories"));
    }

    Ok(())
}

fn validate_file_path_params(params: &Value) -> Result<(), String> {
    if let Some(path) = params.get("path").and_then(|v| v.as_str()) {
        validate_path_security(path)?;
    }
    if let Some(paths) = params.get("paths").and_then(|v| v.as_array()) {
        for path_val in paths {
            if let Some(path) = path_val.as_str() {
                validate_path_security(path)?;
            }
        }
    }
    Ok(())
}
```
